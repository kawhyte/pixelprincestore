## Image Types Overview

The system uses three types of images:

| Type | Location | Format | Purpose |
|------|----------|--------|---------|
| **Card Previews** | `public/art-previews/card/` | WebP | Gallery thumbnails |
| **Detail Images** | `public/art-previews/detail/` | WebP | Product page hero images |
| **Download Files** | `private/free/` | PNG | Actual files users download |

## Preview Images (Gallery Cards)

### Specifications

- **Dimensions**: 600 × 800 pixels
- **Aspect Ratio**: 3:4 (portrait)
- **Format**: WebP (for optimal web performance)
- **Quality**: 80-90% (balance quality and file size)
- **Target File Size**: 50-150 KB
- **Color Space**: sRGB
- **Purpose**: Fast-loading thumbnails for gallery view

### Location

```
public/art-previews/card/
├── ethereal-dreams.webp
├── vintage-map.webp
├── zen-garden.webp
└── botanical-study.webp
```

### Creating Card Previews

**From High-Res Art (Recommended)**:

1. Open your high-resolution artwork
2. Resize to 600 × 800 pixels (maintain aspect ratio)
3. Crop to 3:4 if needed
4. Apply subtle sharpening filter
5. Export as WebP (quality: 85%)
6. Verify file size < 150 KB

**Using ImageMagick (Command Line)**:
```bash
magick input.png -resize 600x800^ -gravity center -extent 600x800 -quality 85 output.webp
```

**Using Online Tools**:
- [Squoosh.app](https://squoosh.app/) - Google's image optimizer
- [CloudConvert](https://cloudconvert.com/) - Batch conversion

### Best Practices

✅ **Do**:
- Use high-quality source images
- Maintain consistent aspect ratios
- Test how images look on both light/dark backgrounds
- Compress without visible quality loss
- Preview on mobile devices

❌ **Don't**:
- Use placeholder images in production
- Exceed 200 KB file size (slow loading)
- Use different aspect ratios between artworks
- Over-compress (creates artifacts)

## Detail Images (Product Pages)

### Specifications

- **Dimensions**: 1200 × 1600 pixels
- **Aspect Ratio**: 3:4 (portrait)
- **Format**: WebP
- **Quality**: 85-95% (higher quality for detail view)
- **Target File Size**: 200-400 KB
- **Color Space**: sRGB
- **Purpose**: High-quality hero images on detail pages

### Location

```
public/art-previews/detail/
├── ethereal-dreams.webp
├── vintage-map.webp
├── zen-garden.webp
└── botanical-study.webp
```

### Creating Detail Images

**From High-Res Art (Recommended)**:

1. Open your high-resolution artwork
2. Resize to 1200 × 1600 pixels
3. Crop to 3:4 if needed
4. Apply subtle sharpening filter
5. Export as WebP (quality: 90%)
6. Verify file size < 500 KB

**Using ImageMagick (Command Line)**:
```bash
magick input.png -resize 1200x1600^ -gravity center -extent 1200x1600 -quality 90 output.webp
```

### Optional: Fallback to Card Preview

If you don't provide a detail image, the system automatically uses the card preview. However, providing a higher-resolution detail image improves the user experience.

**In config/free-art.ts**:
```typescript
{
  previewImage: "/art-previews/card/ethereal-dreams.webp",
  detailImage: "/art-previews/detail/ethereal-dreams.webp", // Optional
}
```

## Downloadable Art Files

### Specifications

These are the actual PNG files users download for printing.

| Size | Dimensions | DPI @ Size | File Format | Target File Size |
|------|------------|------------|-------------|------------------|
| **4" × 5"** | 1200 × 1500 px | 300 DPI | PNG | 1-2 MB |
| **8" × 10"** | 2400 × 3000 px | 300 DPI | PNG | 2-4 MB |
| **16" × 20"** | 4800 × 6000 px | 300 DPI | PNG | 7-10 MB |
| **40 × 50 cm** | 4724 × 5906 px | 300 DPI | PNG | 7-10 MB |

### Location

```
private/free/
├── ethereal-dreams-4x5.png
├── ethereal-dreams-8x10.png
├── ethereal-dreams-16x20.png
├── ethereal-dreams-40x50cm.png
├── ethereal-dreams-all.zip          ← Generated by script
├── vintage-map-4x5.png
├── vintage-map-8x10.png
├── vintage-map-16x20.png
├── vintage-map-40x50cm.png
├── vintage-map-all.zip               ← Generated by script
... (etc)
```

### Creating Download Files

**Print-Ready Requirements**:
- ✅ 300 DPI minimum
- ✅ PNG format (lossless, supports transparency)
- ✅ sRGB color space (for consistent printing)
- ✅ No watermarks or logos
- ✅ Full bleed if designing for frames

**From Vector/High-Res Source**:

1. Open your artwork in design software
2. Export at exact dimensions listed above
3. Use 300 DPI export setting
4. Save as PNG-24 (24-bit color + alpha)
5. Verify file size is reasonable
6. Test print at smallest size to verify quality

**Using ImageMagick (Batch Export)**:
```bash
# From a very high-res source
magick source.png -resize 1200x1500 -density 300 ethereal-dreams-4x5.png
magick source.png -resize 2400x3000 -density 300 ethereal-dreams-8x10.png
magick source.png -resize 4800x6000 -density 300 ethereal-dreams-16x20.png
magick source.png -resize 4724x5906 -density 300 ethereal-dreams-40x50cm.png
```

### After Adding Files

Run the ZIP generation script to create download bundles:
```bash
npm run generate-zips
```

This creates ZIP files with all 4 sizes + HOW-TO-OPEN.txt instructions.

## Optimization Guidelines

### Compression Tools

**Command Line**:
- **ImageMagick**: `magick convert input.png -quality 85 output.webp`
- **cwebp** (Google): `cwebp -q 85 input.png -o output.webp`

**GUI Applications**:
- **Squoosh** (Web): https://squoosh.app/
- **XnConvert** (Desktop, Free): Batch processing
- **Photoshop**: Export As → WebP

**Automated Workflows**:
```bash
# Batch convert all PNGs to WebP
for file in *.png; do
  cwebp -q 85 "$file" -o "${file%.png}.webp"
done
```

### Performance Targets

| Image Type | Max File Size | Load Time Target |
|------------|---------------|------------------|
| Card Preview | 150 KB | < 0.5s (fast 3G) |
| Detail Image | 400 KB | < 1.0s (fast 3G) |
| Download File | No limit | N/A (direct download) |

### Testing Tools

- **Lighthouse** (Chrome DevTools): Performance audit
- **WebPageTest**: Real-world speed testing
- **ImageOptim** (Mac): Lossless compression

## File Naming Conventions

### Preview Images

**Pattern**: `{artwork-id}.webp`

Examples:
- `ethereal-dreams.webp`
- `vintage-map.webp`
- `zen-garden.webp`
- `botanical-study.webp`

**Rules**:
- Lowercase only
- Use hyphens (not underscores or spaces)
- Match the `id` field in config (without `art_` prefix)
- Use `.webp` extension

### Download Files

**Pattern**: `{artwork-id}-{size-id}.png`

Examples:
- `ethereal-dreams-4x5.png`
- `ethereal-dreams-8x10.png`
- `vintage-map-16x20.png`
- `zen-garden-40x50cm.png`

**Rules**:
- Lowercase only
- Use hyphens to separate parts
- Must match `fileName` in config
- Use `.png` extension
- Size IDs: `4x5`, `8x10`, `16x20`, `40x50cm`

### ZIP Files

**Pattern**: `{artwork-id}-all.zip`

Examples:
- `ethereal-dreams-all.zip`
- `vintage-map-all.zip`

**Generated automatically** by `npm run generate-zips`

## Example Workflow

### Step-by-Step: Adding "Ethereal Dreams"

1. **Prepare Source**:
   - Open high-resolution artwork (minimum 4800×6000px)
   - Verify color space is sRGB
   - Save as lossless format (PNG or TIFF)

2. **Create Download Files**:
   ```bash
   # Export 4 sizes at 300 DPI
   magick source.png -resize 1200x1500 -density 300 ethereal-dreams-4x5.png
   magick source.png -resize 2400x3000 -density 300 ethereal-dreams-8x10.png
   magick source.png -resize 4800x6000 -density 300 ethereal-dreams-16x20.png
   magick source.png -resize 4724x5906 -density 300 ethereal-dreams-40x50cm.png

   # Move to private/free/
   mv ethereal-dreams-*.png private/free/
   ```

3. **Create Web Previews**:
   ```bash
   # Card preview (600×800)
   magick source.png -resize 600x800^ -gravity center -extent 600x800 -quality 85 card.webp
   mv card.webp public/art-previews/card/ethereal-dreams.webp

   # Detail image (1200×1600)
   magick source.png -resize 1200x1600^ -gravity center -extent 1200x1600 -quality 90 detail.webp
   mv detail.webp public/art-previews/detail/ethereal-dreams.webp
   ```

4. **Generate ZIP**:
   ```bash
   npm run generate-zips
   # Creates: private/free/ethereal-dreams-all.zip
   ```

5. **Verify in Browser**:
   - Open http://localhost:3000/free-downloads
   - Check card displays correctly
   - Click card → verify detail page
   - Test download single size
   - Test download ZIP bundle

## Troubleshooting

### Issue: Images Not Displaying

**Symptom**: Broken image icons or alt text showing

**Solutions**:
1. Verify file exists at exact path specified in config
2. Check file naming (case-sensitive, no typos)
3. Ensure WebP images are in `public/` directory
4. Clear Next.js cache: `rm -rf .next`
5. Restart dev server: `npm run dev`

### Issue: Images Look Blurry

**Symptom**: Soft, pixelated, or low-quality appearance

**Solutions**:
1. Increase source resolution before resizing
2. Don't upscale images (always downscale from high-res)
3. Apply sharpening filter after resize
4. Increase WebP quality setting (try 90-95%)
5. Use PNG for detail images if WebP compression is too aggressive

### Issue: File Sizes Too Large

**Symptom**: Slow page loading, high bandwidth usage

**Solutions**:
1. Reduce WebP quality (try 80% instead of 90%)
2. Resize to exact specifications (don't exceed recommended dimensions)
3. Use tools like Squoosh or ImageOptim for better compression
4. Consider AVIF format for even better compression (if browser support allows)
5. Enable Next.js image optimization (automatic with next/image component)

### Issue: Colors Look Wrong

**Symptom**: Colors appear washed out, oversaturated, or different than source

**Solutions**:
1. Convert to sRGB color space before exporting
2. Disable color profile embedding
3. Use consistent color space across all files
4. Test on multiple devices/browsers
5. Use color calibrated monitor for editing
